Principais Tecnologias

Nodejs

Baseado no interpretador V8, o NodeJS viabiliza a execução de código javascript no lado do servidor (beckend)

Umas das tecnologias que tenho mais utilizado nos últimos anos para projetos de estudo e soluções reais. Desde a criação de REST APIs — que frequentemente integro com ReactJS — quanto para monolitos geralmente acompanhados do simplista EJS.

ReactJS

Uma biblioteca de javascript que permite criar interfaces de usuário para páginas web (frontend).

O ReactJS viabiliza a criação de páginas SPAs (ou Single Page Applications) tendo assim, páginas mais rápidas e parecindas com aplicações pré-instaladas.

A codificação se torna rápida organizada devido a suas características, uma delas é a componentização de elementos que podem ser reaproveitados em várias partes da página ou aplicação, além de unir html à interações javascript em arquivos únicos.

Utilizo-o sempre que possível, desde que atenda as necessidades do projeto em questão, assim entregando um software de maior qualidade em um tempo relativamente mais curto.

Typescript

Superset de javascript, o qual adiciona tipagem estática à codificação. Podendo ser utilizado no lado frontend e/ou backend, assim facilitando um código fonte mais organizado e limpo, que por sua vez reduz chances de haver inconsistências na execução do software.

E uma vez inserido nos meus projetos, nunca mais ficou de fora. Pois que por mais que exista uma curva de aprendizagem inicial, me garantiu maior segurança de desenvolvimento, quase que inibindo por completo bugs e imprevistos relacionados a tipagem e a arquitetura do código.

HTML, CSS e Javascript

Linguagem de marcação, estilização e programação, respectivamente. Utilizados na criação de páginas web, inserindo conteúdo, estilo visual e interação para com o usuário.

Utilizo-os de forma "pura" (endenda-se sem frameworks ou super bibliotecas) ainda para pequenas páginas web ou landing pages independentes e estáticas. Pois não necessita de grandes configurações inicias no projeto.

Contudo, mesmo utilizando pouco de forma "pura", todo o conhecimento é reaproveitado em outras construções de software aqui já citadas.
